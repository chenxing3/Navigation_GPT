# Navigation_GPT

### A bat‑inspired GPT that self‑learns to map and navigate large‑scale environments from acoustic information

This repository contains the code, data, and figure scripts for the study:

> **A bat‑inspired GPT self‑learns to map and navigate large‑scale environments based on acoustic information**
> Xing Chen and Yossi Yovel, School of Zoology & Sagol School of Neuroscience, Tel Aviv University.

---

## Overview

Animals navigate complex environments by building an internal spatial map from raw
sensorimotor experience. In echolocating bats, this map can be **echoic**. Here we ask
whether a general **sequence‑learning** model can learn such a map from scratch — with **no
position supervision and no reward** — using only self‑movement and echoes.

We train an autoregressive, decoder‑only Transformer (**GPT‑Neo backbone**) to predict a
simulated bat's next movement and sensory landmark from the preceding stream of movement
and echo tokens, while it "flies" through a kilometre‑scale 3‑D reconstruction of the Hula
Valley (Israel). After training, the model can:

- generate accurate long‑range trajectories (up to ~10 km) between arbitrary locations,
  including **novel short‑cuts** never seen during training;
- **recalibrate** its path‑integrated position at acoustic landmarks (sensory locations, SLs)
  by classifying their echoes;
- develop an emergent internal representation with **place‑like, border‑like, and
  head‑direction‑like** tuning, and a 1‑D ring manifold ("internal compass") — none of which
  were explicitly programmed.

The model learns the **grammar of navigation** the same way a language model learns the grammar
of text: by next‑element prediction over an ordered stream of movements, locations, and echoes.

## How it works

### The "language of navigation"

Trajectories are encoded as a mixed sequence of three token types:

| Token | Meaning |
|-------|---------|
| **Movement tokens** | idiothetic (self‑motion) cues; BPE‑merged sequences of 5 m azimuth steps (~10 m per token on average) |
| **Location tokens (SLs)** | discrete sensory locations that can serve as landmarks (1 ID per zone) |
| **Echo tokens** | the acoustic snapshot at a sensory location; six raw echoes encoded to a 1024‑D vector |

A trajectory is reconstructed from movement tokens by stepping 5 m along each azimuth
(`Δx = 5·sin θ`, `Δy = 5·cos θ`), and SL/echo tokens are interleaved wherever the bat passes a
landmark.

### Architecture

The model has two branches that share a common representation:

1. **Sensory network (echo encoder).** A 3‑layer 1‑D CNN decomposes six raw echo waveforms,
   a 2‑layer Transformer encoder adds context, and a single learnable query attends over the
   result to produce one **1024‑D sensory vector** per landmark.
2. **Navigational network (GPT).** A **24‑layer GPT‑Neo** (HuggingFace), with the **positional
   encodings removed** so that all spatial structure is learned from token content rather than
   token index. It predicts the next movement token and, near a landmark, the location ID.
3. **Read‑out probes.** Small MLPs trained on the *frozen* hidden state for evaluation only
   (e.g. distance / heading inference, SL classification).

All weights are **randomly initialised** (no text‑pretrained weights).

## Data

The environment is a 3‑D point‑cloud reconstruction of the **Hula Valley** (10 × 8 km²).

- **1,532 sensory zones** (60 m radius), each sampled at 4–6 points → **9,422 sensory locations
  (SLs)** with a 30 m radius.
- **~14.4 M simulated echoes** (30–48 kHz, single ear), generated with the sonar equation from
  beam directionality, landscape geometry, and acoustic attenuation, with added Gaussian and
  correlated noise.
- **~9.1 M flight paths** generated by a correlated random walk matched to real Kuhl's
  pipistrelle statistics (median speed 5.4 m/s; Von Mises turning angles).

Generated trajectories are stored as `.feather` tables (one row per trajectory) containing,
among others: `loc1_x/loc1_y` (origin), `loc2_x/loc2_y` (destination),
`azimuths_tokens_str` (movement tokens), `full_generated_sequence` (interleaved tokens),
`full_sequence_entropies` (per‑token next‑token entropy), and `middle_locations_texts`
(recalibration SLs).

## Training

Training is a two‑stage, **self‑supervised** procedure (no ground‑truth position labels).

**1) Pre‑training — learn the map.**
The model is trained on long, goal‑less sequences of movement and interleaved SL/echo tokens,
with two concurrent objectives:
- *movement prediction* — next‑token prediction over the movement/SL stream;
- *sensory localization* — classify a landmark's ID from a sequence of five echo vectors.

Informative landmarks are sampled more often via an adaptive echo‑loss weighting scheme.

**2) Fine‑tuning — learn to plan.**
The pre‑trained model is trained on goal‑directed sequences that begin with an
`(origin, destination)` pair, followed by a high‑level path plan (a sequence of intermediate
SLs) and the low‑level movement sequence.

```bash
# ILLUSTRATIVE — replace with your actual entry points / config
python train/pretrain.py    --config configs/pretrain.yaml
python train/finetune.py    --config configs/finetune.yaml --init checkpoints/pretrained.pt
```

A **partial‑training** variant (pre‑train on part of the map, then fine‑tune the final layers on
a new region with only 1 Mm of trajectories) is also included, to probe data‑efficient transfer.

## Testing / evaluation

Given an `(origin, destination)` pair the model generates a full flight path:

1. **Plan** — sample a sequence of intermediate SLs (top‑p / top‑k sampling).
2. **Execute** — roll out the low‑level movement tokens, tracking position by path integration.
3. **Recalibrate** — on entering a landmark's 30 m radius, classify its echoes; if confident
   (softmax > 0.5) and consistent with path integration, continue, otherwise reset to the last
   verified landmark and re‑plan (bounded number of attempts).

```bash
# ILLUSTRATIVE — replace with your actual entry points / config
python test/generate_trajectories.py --checkpoint checkpoints/finetuned.pt --out dataset/trajectory/...
python test/evaluate.py              --results dataset/trajectory/.../trajectories_*.feather
```

Reported metrics include endpoint **distance error**, **completion rate** (fraction reaching
within 100 m of the target), **geometric inference** accuracy (MLP probes for pairwise distance
and allocentric angle), and per‑location **echo‑localization** accuracy.

## Reproducing the figures

Scripts under `Figure_1/` reconstruct trajectories from the `.feather` results and render the
paper figures (using `ops/token_info.csv` for landmark coordinates). For example, a single
trajectory can be plotted by selecting its row index from a results file.
